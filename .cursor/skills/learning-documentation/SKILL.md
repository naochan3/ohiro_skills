# Learning Documentation

## When to Use
- 体系的にプロジェクト知識を残したい時

## Instructions
1. **骨子を作る**
   - 概要 / 技術スタック / フォルダ構成 / 処理フロー / 変更ログ
2. **図と例を追加**
   - 重要なフローは簡単な図で可視化
3. **更新ルールを決める**
   - 変更時に必ず追記する項目を明記
