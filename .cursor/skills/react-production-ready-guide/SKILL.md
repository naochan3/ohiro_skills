# React Production Ready Guide

## いつ使うか
- Reactアプリの新規設計/リファクタリング
- コンポーネント分割や状態管理の方針決定
- パフォーマンス/品質劣化を防ぎたい時

## 参照ルール（最優先）
- `.cursor/rules/05-react-patterns.mdc`
- `.cursor/rules/11-code-hygiene-and-refactoring.mdc`
- `.cursor/rules/27-testing-strategy.mdc`

## 要点（統合）
- 機能単位の構成と境界を明確にする
- プレゼンテーショナル/コンテナ分離を徹底する
- 典型的なReactアンチパターンを避ける
- エラーハンドリングとテスト戦略を前提に設計する

## 手順
1. 画面/機能単位でフォルダ構成を決める
2. コンポーネントの責務を分離する
3. 既知のアンチパターンを排除する
4. 状態管理/データ取得/エラーハンドリングを設計
5. テスト戦略（Unit/Integration/E2E）を明記

## アンチパターンチェック
- propsを初期stateへコピーしていない
- indexをkeyに使っていない
- 1コンポーネントに責務が詰まりすぎていない
- 不要な再レンダリングが起きていない

## 出力テンプレ
1. フォルダ構成（機能単位）
2. コンポーネント責務の分割方針
3. 状態管理/データ取得の方針
4. エラー/ローディングの扱い
5. テスト戦略
