# Doc Co-Authoring

ドキュメントを「情報収集 → 構成整理 → 仕上げ/検証」で共著するための手順です。

## いつ使うか

- 仕様書、RFC、提案書、決定記録、設計書の作成
- 既存ドキュメントの再構成/改善

## 実行手順

### 1. コンテキスト収集
1. 文書タイプ、対象読者、期待効果を確認する
2. テンプレや既存資料の有無を確認する
3. 事実関係・制約・関連リンクを「情報ダンプ」で集める
4. 不明点を5〜10問で質問し、抜けを埋める

### 2. 構成整理
1. セクション案を3〜6個提示し、合意する
2. 章ごとに「含める要素」を箇条書きで整理する
3. 読者の疑問の順番に並べ替える

### 3. 共同執筆
1. 章ごとに下書きを作成
2. レビュー → 修正 → 再レビューを繰り返す
3. 主語/目的/決定/根拠が一目でわかるように整える

### 4. Reader Testing
1. 初見の読者が誤解しそうな点を列挙
2. その点を先回りで補足/修正
