# Solo Dev Workflow

## When to Use
- 個人開発の立ち上げ時
- ワークフローを簡略化したい時

## Instructions
1. **最短で動く土台を作る**
   - 必要最低限の構成と自動化のみ先に用意
2. **CI/CDの初期化**
   - 失敗時の診断手順を用意する
3. **タイムボックス運用**
   - 作業時間と検証時間を明確に区切る
