# Design System Generator and Validator

## いつ使うか
- UI/UX設計やLP制作の初期方針決定
- デザインシステムの新規作成/刷新
- デザイン受領時の品質チェック（handoff）

## 参照ルール（最優先）
- `.cursor/rules/01-design-system.mdc`
- `.cursor/rules/26-implementation-process.mdc`
- `.cursor/rules/45-image-generation-flow.mdc`

## 要点（統合）
- 業種/用途からデザインシステムを逆算する（トーン/信頼/速度感）
- スタイルは「目的に必然性があるものだけ」採用する
- 業界ごとのアンチパターンを除外する（誤解や不信を生む表現）
- デザインハンドオフは「状態・レスポンシブ・素材」を網羅する
- デザインシステムはトークン/ガイド/コンポーネントで文書化する

## 手順
1. 目的/業界/主要CTA/利用環境（B2B/B2C/端末）を整理
2. トーンとビジュアル方針（色/タイポ/余白/影）を決定
3. NG表現の除外（業界不適合・AIっぽさ・過剰演出）
4. コンポーネントと状態（hover/focus/disabled/error）を定義
5. ハンドオフチェックで不足素材/状態を洗い出す

## ハンドオフチェックリスト
- グリッド仕様（カラム/ガター/最大幅）が明記されている
- カラートークンの命名と用途が揃っている
- フォント形式/ウェイト/サイズの提供がある
- リンク/ボタン/フォームの状態が定義されている
- アイコンはSVGで、規格（サイズ/線幅）が揃っている
- 主要ブレークポイントのデザインがある
- 画像/イラストの利用方針が明記されている

## 出力テンプレ
1. 目的 / 業界 / 主要CTA
2. デザイン方針（色/タイポ/レイアウト/影）
3. NGパターン（避ける表現）
4. コンポーネントと状態一覧
5. ハンドオフ不足リスト
