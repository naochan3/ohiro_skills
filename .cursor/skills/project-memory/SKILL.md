# Project Memory

## When to Use
- 重要な仕様判断、設計変更、方針変更があった時

## Instructions
1. **記録先の確保**
   - `docs/memory/` を用意し、日付ベースでファイルを作る
2. **要点の記録**
   - 何を決めたか / なぜそうしたか / 代替案は何だったか
3. **影響範囲の記録**
   - 影響する画面・API・データモデルを明記
4. **次のアクション**
   - フォローアップタスクがあれば箇条書きで残す
