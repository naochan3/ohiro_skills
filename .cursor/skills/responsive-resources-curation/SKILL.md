# Responsive Resources Curation

## いつ使うか
- レスポンシブUIの参考例を集めたい時
- 画面構成やレイアウトの方向性を固めたい時
- 既存のUI/UXリソースだけでは足りない時

## 参照ルール（最優先）
- `.cursor/rules/01-design-system.mdc`
- `.cursor/rules/23-security-policy.mdc`

## 実行手順
1. **目的整理**: 対象画面/ユースケース/主要デバイスを明確化する
2. **収集**: リソースから該当しそうなパターンを3〜7件抽出する
3. **評価**: アクセシビリティ/可読性/情報設計/パフォーマンス観点で比較する
4. **選定**: 最終候補を1〜3件に絞り、採用理由を言語化する
5. **落とし込み**: 自プロジェクトのデザインルールへ翻訳して適用する

## チェックリスト
- モバイル/タブレット/デスクトップの優先度が明確
- 情報設計の差分（要約/詳細）が整理できている
- 参考例のライセンスや利用条件が確認済み

## 出力テンプレ
1. 目的/対象画面
2. 参考候補（3〜7件）
3. 評価結果（良い点/懸念点）
4. 採用方針（1〜3件）
5. プロジェクトへの適用メモ
