# Event-Driven Microservices Playbook

## いつ使うか
- マイクロサービス化を検討する時
- イベント駆動アーキテクチャを導入する時
- サービス間通信や観測性を設計する時

## 参照ルール（最優先）
- `.cursor/rules/28-observability-and-logging.mdc`
- `.cursor/rules/30-non-functional-requirements.mdc`

## 要点（統合）
- 非同期は整合性モデル（即時/最終）を前提に設計する
- サービス境界は「変更理由」で切る
- サービスメッシュは段階導入と運用設計が必須

## 手順
1. サービス境界と責務を定義
2. イベントの命名規約とスキーマを決定
3. ブローカ/フレームワーク選定（例: Eventuate/Axon）
4. 監視・トレース・リトライ設計を行う
5. 必要ならサービスメッシュを段階導入

## チェックリスト
- 整合性モデルが明確
- 冪等性の考慮がある
- リトライ/デッドレターの方針がある
- トレース/メトリクスの設計がある
- サービスメッシュ導入の目的が明確

## 出力テンプレ
1. サービス境界と責務
2. イベント一覧（発火条件/ペイロード）
3. ブローカ/フレームワーク選定理由
4. 運用設計（監視/リトライ/追跡）
