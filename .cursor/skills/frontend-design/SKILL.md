# Frontend Design

文脈に合った高品質なUIを設計・実装するための手順です。  
**プロジェクト内のデザインルールが最優先**です。

## いつ使うか

- 新規画面/コンポーネントの設計
- UIの品質向上や再設計

## 実行手順

1. **目的/対象/制約の定義**:
   - 目的、主要導線、ユーザー像、制約（技術/パフォーマンス）を整理
2. **トーン選定**:
   - クリエイティブ or 実用モードを判定
   - トーンを1つに絞る（例: ミニマル/編集的/ラグジュアリー）
3. **タイポグラフィ設計**:
   - 見出し/本文の階層とサイズを決める
4. **配色/コントラスト**:
   - ベース/アクセントの比率を固定
   - WCAG AAのコントラストを確保
5. **レイアウト/余白**:
   - グリッドとリズムを決める
   - 情報の優先順位が一目で分かる配置にする
6. **インタラクション**:
   - 必要なホバー/フォーカス/ローディング状態を定義
7. **実装と検証**:
   - 画面を組み立て、アクセシビリティと再利用性を確認
