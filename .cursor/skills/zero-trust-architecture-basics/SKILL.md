# Zero Trust Architecture Basics

## いつ使うか
- ゼロトラスト導入を検討する時
- ネットワーク境界ではなく「継続検証」を前提に設計する時

## 要点
- 暗黙の信頼を排除し、常に検証する
- 最小権限を徹底する
- 継続的な監視と評価を前提にする

## 手順
1. 資産/データ/ユーザーの境界を棚卸し
2. 認証/認可の強化（多要素/条件付きアクセス）
3. デバイス/コンテキストによるアクセス制御
4. ログ/監視/検知体制の整備

## チェックリスト
- データ分類ができている
- 重要システムへのアクセスが最小権限
- 例外アクセスの監査が可能
- 監視とアラートが継続的に回る

## 出力テンプレ
1. 対象資産と保護レベル
2. 認証/認可方針
3. 監視/検知設計
4. 移行ステップ
